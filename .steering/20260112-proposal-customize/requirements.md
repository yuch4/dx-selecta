# 要件定義: 提案書カスタマイズ機能

## PRDからの抽出

### ユーザーストーリー
情シス担当として、上司・経営層に提案するために、比較結果から稟議書を自動生成し、**必要に応じてカスタマイズ**したい

### 受け入れ条件（PRDより）
- [x] Markdown形式で稟議書を生成・表示できる（実装済み）
- [ ] **テンプレート項目をカスタマイズできる**（未実装）
- [ ] 生成履歴が保存される（部分実装）
- [ ] 条件変更後の再生成ができる（部分実装）
- [ ] 生成完了まで5秒以内（実装済み）

### GAP分析での指摘
- 現在の実装: 自動生成のみ、編集不可
- 必要な機能: セクション編集、テンプレートカスタマイズ

## 機能要件

### 1. セクション編集機能
- 各セクション（背景、目的、比較、推奨理由など）を個別に編集可能
- インライン編集UI（クリックで編集モード）
- 編集後のプレビュー表示

### 2. カスタムセクション追加
- ユーザー定義のセクションを追加可能
- セクション順序の入れ替え

### 3. テンプレート管理
- デフォルトテンプレートの提供
- ユーザーカスタムテンプレートの保存
- テンプレート選択機能

### 4. バージョン管理
- 編集履歴の保持
- 過去バージョンへの復元

## 優先順位

**MVP（今回実装）**
1. セクション編集機能（インライン編集）
2. 編集内容の保存・反映
3. 編集履歴（バージョン番号管理）

**Post-MVP**
- カスタムセクション追加
- テンプレート管理
- 複雑なバージョン管理

## 技術的考慮事項

- Markdownのセクション解析（##で分割）
- textarea vs contenteditable
- Supabaseでの履歴管理（既存proposal_outputs拡張）
