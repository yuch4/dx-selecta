# 案件応募PR文生成

## 最重要ルール

**職務経歴書に記載された事実のみを使用すること。**
- 推測・創作・誇張は一切禁止
- 該当経験がなければ「該当なし」と正直に記載
- PR文の各記述は職務経歴書から出典を示せること

## 入力

募集要項:
$ARGUMENTS

## 処理

1. `job-application-writer` スキルを読み込む（絶対厳守ルールを確認）
2. 職務経歴書データを**必ず読み込む**:
   - `フリーランス/職務経歴書.md`
   - `フリーランス/職務経歴書_自己アピール_情シス向け.md`
3. 募集要項を分析し、マッチング評価を実施
   - **各項目の該当経験は職務経歴書の出典を明記**
   - **該当なしの項目も正直に記載**
4. PR文を全バリエーション（100字/200字/400-600字/800-1000字）で生成
   - **生成後、各記述が職務経歴書に存在するかセルフチェック**

## 出力

スキルの出力フォーマットに従い、以下を出力:
- マッチング分析結果（出典付き）
- PR文（4バリエーション）
- カスタマイズ提案

## 禁止事項チェックリスト

生成後、以下を確認:
- [ ] 職務経歴書にないツール・技術名を記載していないか
- [ ] 職務経歴書にない役職・プロジェクトを記載していないか
- [ ] 数値（人数、期間、規模）を誇張していないか
- [ ] 「〜もできる」「〜にも対応可能」等の推測を含んでいないか
