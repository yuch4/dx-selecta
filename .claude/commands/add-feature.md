# 新機能追加

ステアリング駆動の新機能追加ワークフロー。各フェーズで承認を得てから次に進む。

## 入力

機能要件: $ARGUMENTS

---

## ワークフロー

### Phase 1: 計画（承認必須）

1. **要件の明確化**
   - 機能の目的とゴール
   - 対象ユーザーとユースケース
   - 成功基準

2. **設計検討**
   - 技術的アプローチ（複数案提示）
   - 影響範囲の分析
   - リスクと対策

3. **タスクリスト作成**
   - 実装ステップを分解
   - 各タスクの見積もり
   - 依存関係の整理

**出力**: 計画書（Markdown形式）

**ユーザー承認を待つ**
→ 承認なしで実装に進まないこと

---

### Phase 2: 実装（タスクループ）

承認後、タスクリストに従って実装:

```
for each task in タスクリスト:
    1. タスク開始を報告
    2. 実装
    3. セルフレビュー
    4. タスク完了を報告
    5. 次のタスクへ
```

**実装ルール**:
- 1タスクずつ確実に完了
- 変更は最小限に留める
- テストを書く（該当する場合）
- コミットメッセージは明確に

---

### Phase 3: 検証

1. **動作確認**
   - 手動テスト
   - 自動テスト実行

2. **品質チェック**
   - リントエラーなし
   - 型エラーなし
   - セキュリティ考慮

3. **レビュー準備**
   - 変更サマリ
   - テスト結果
   - 残課題（あれば）

---

### Phase 4: 完了報告

**出力**:
```markdown
## 実装完了レポート

### 実装内容
- 追加した機能
- 変更したファイル

### テスト結果
- 実行したテスト
- 結果

### 次のステップ
- PR作成 or マージ準備
- 追加で必要な作業
```
