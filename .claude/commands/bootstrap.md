# Bootstrap

ユーザーの要望を分析し、既存資産を最大限活用した実現プランを提案する。

## 入力
$ARGUMENTS（何を実現したいか）

## 処理手順

### Step 1: 要望の分解
- 要望を具体的なタスク・成果物に分解
- 必要な機能・技術要件を特定

### Step 2: 既存スキルのスキャン
以下をスキャンして活用可能な資産を探す:
- `.claude/skills/*/SKILL.md` - スキルの description を確認

### Step 3: マッチング結果を提示

| 種類 | 名前 | 用途 | マッチ度 |
|------|------|------|---------|
| skill | 名前 | 説明 | ⭐⭐⭐ |

新規作成が必要な場合:

| 種類 | 提案名 | 理由 |
|------|--------|------|
| skill | 名前 | なぜ必要か |

### Step 4: アクションプラン提示
1. ステップ1: ...
2. ステップ2: ...

### Step 5: 実行（承認後）
- 既存スキルを読み込んで指示に従う
- 新規作成時は skill-creator スキルを参照

## 出力フォーマット

```markdown
## 🔍 要望の理解
（分解結果）

## 📊 スキルマッチング
（既存スキル・新規作成の表）

## 🚀 アクションプラン
（ステップ）

## ❓ 確認事項
（あれば）
```
