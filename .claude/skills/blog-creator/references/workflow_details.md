## C5: サムネイル画像プロンプトのYAMLフォーマット

### 出力フォーマット（05_thumbnail_prompts.md）

```markdown
## C5: メインサムネイル生成プロンプト（YAML形式）

### 記事分析とコピーライティング
- **コアメッセージ**: [記事の中心的なメッセージ]
- **プロデザイナー基準のコピー案**:
  - 案A用: [テーマ名] + [インパクト]
  - 案B用: [テーマ名] + [信頼性]
  - 案C用: [テーマ名] + [親しみ]
  - 案D用: [テーマ名] + [ユーザー指定スタイル]

### メインサムネイル案

**案A: インパクト重視型**
```yaml
画像生成設定:
  背景: "純白背景"
  スタイル: "インパクト・アイキャッチ"
  デザイン: "ミニマル・フラットデザイン"

主要要素:
  - 要素1: "[テーマを象徴する具体的モチーフ]"

日本語テキスト:
  メインタイトル: "[テーマ名（必須）]"
  インパクト数値: "[ベネフィット]"

文字化け対策:
  フォント指定: "Noto Sans JP Bold"
  エンコーディング: "UTF-8"
  レンダリング: "高品質"

構図:
  レイアウト: "中央配置（上下に十分な余白を確保）"
  視覚的階層: "タイトル → モチーフ → 背景"
  SNS最適化: "16:9・セーフエリア考慮"
```

**案B: 信頼性重視型**
```yaml
（案Aと同様の構造で、プロフェッショナルな表現）
```

**案C: 親しみやすさ重視型**
```yaml
（案Aと同様の構造で、温かみを重視）
```

**案D: ユーザースタイル（※style.mdがある場合のみ生成）**
```yaml
（user_styleで指定されたトーン＆マナーを反映した案）
```

## 【オプション】Antigravity Nano Banana Pro用 追加ルール
<!-- Antigravity環境で生成する場合（C5-1）に、プロンプト末尾へ付与する -->
- 正方形キャンバスの横幅いっぱいに合わせた、明確な16:9ワイド比率の画像を生成すること
- 制約: 上下のエリアは必ず真っ白な空白（ホワイトレターボックス）
- 禁止: 正方形キャンバス全体を縦に埋め尽くすこと禁止。上下余白部分にオブジェクト、文字、背景を配置すること禁止
- 例: 1024x1024の場合は中央1024x576のみ描画し、上下は純白固定
```

### 案A〜Cの方針

| 案 | 方向性 | 配色 | モチーフ | メッセージ |
|----|--------|------|----------|------------|
| A | インパクト | 鮮やか（赤、オレンジ、黄） | 大胆・象徴的 | 数値・ベネフィット前面 |
| B | 信頼性 | 落ち着き（濃紺、グレー、ゴールド） | 図解・グラフ的 | 権威性・専門性 |
| C | 親しみ | 柔らか（パステル、アースカラー） | イラスト的・温かみ | 共感・寄り添い |

### 案Dの扱い

`references/image_style/style.md` が空でない場合のみ作成。ユーザー指定のスタイル（企業ブランディング、特定のビジュアル言語等）に基づいた案。

## 画像ファイルの命名規則

### サムネイル画像

```
Image/thumbnail/thumbnail_A_impact.png
Image/thumbnail/thumbnail_B_trust.png
Image/thumbnail/thumbnail_C_friendly.png
Image/thumbnail/thumbnail_D_custom.png  # 案Dがある場合のみ
```

### セクション画像

```
Image/sections/section_01_[slug].png
Image/sections/section_02_[slug].png
...
```

`[slug]` はセクション名から生成した短い英数字識別子（例: `paradigm_shift`, `case_study`）

---

## 再実行時のファイル退避

既存ファイルを上書きする前に、必ず `archive/` に退避。

### 退避名の形式

```
archive/YYYYMMDD_HHMMSS_元のファイル名
```

例:
```
archive/20260110_143012_04_blog_post.md
archive/20260110_143012_05_thumbnail_prompts.md
```

### 退避対象

- C3以降のすべての成果物
- 画像ファイル（再生成時）

### 退避しないもの

- `sources/c2/` 配下（ユーザー管理）
- `references/` 配下（参照用）

---

## 事実の扱いルール

- `sources/c2/` に存在しない事実・数値・引用は**捏造しない**
- 根拠が弱い断定はしない（「仮説」「前提条件付き」で扱う）
- 数値は可能な限り出典URLを残す（C2由来のみ）

---

## 言語ルール

- 成果物はすべて**日本語**
- 画像生成プロンプト（YAML内のテキスト含む）は**日本語のみ**
