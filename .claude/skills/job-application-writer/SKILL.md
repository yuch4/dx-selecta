# Job Application Writer

案件応募時の意気込み・PR文を生成する。

## 絶対厳守ルール（最重要）

### 事実のみ使用の原則

**職務経歴書に明記されている事実のみを使用すること。**

| 許可 | 禁止 |
|------|------|
| 職務経歴書に記載された実績・数値 | 記載されていない実績の創作 |
| 記載されたスキル・ツール名 | 「〜も対応可能」等の推測 |
| 記載された役職・期間 | 曖昧な表現の拡大解釈 |
| 記載された成果の言い換え | 成果の誇張・水増し |

### 禁止事項（違反は嘘になる）

1. **推測の禁止**: 「おそらく〜の経験もある」→ NG
2. **創作の禁止**: 職務経歴書にない業務・ツールを記載 → NG
3. **誇張の禁止**: 「リーダー」を「統括責任者」に変える → NG
4. **補完の禁止**: 「〜の経験があるので〜もできる」→ NG

### 迷ったときの判断基準

**「この記述は職務経歴書のどこに書いてあるか？」**
→ 指し示せなければ記載しない。

### 不足スキルがある場合の対応

- 「該当経験なし」と正直に記載
- 関連する経験があれば「直接の経験はないが、〜の経験あり」と明記
- 学習意欲で補う場合は「現在学習中」等、事実のみ記載

## ワークフロー

### Phase 1: 情報収集

1. 募集要項を取得（ユーザーから提供）
2. 職務経歴書を読み込む: [references/resume-data.md](references/resume-data.md)
3. 募集要項から以下を抽出:
   - 必須スキル・経験
   - 歓迎スキル・経験
   - 業務内容・ポジション
   - 求める人物像

### Phase 2: マッチング分析

1. 職務経歴書のスキル・経験と募集要項を照合
2. **各項目について職務経歴書の該当箇所を明示**
3. マッチ度を算出（高・中・低・該当なし）
4. 強調すべきポイントを特定（出典明記）
5. 不足スキル・ギャップを正直に特定

**重要**: マッチ度評価時、該当経験がない場合は「該当なし」と明記。

### Phase 3: PR文生成

[references/templates.md](references/templates.md) のテンプレートを参考に、以下を生成:

1. **超短縮版**（100文字）: 一言PR
2. **短縮版**（200文字）: SNS・チャット向け
3. **標準版**（400-600文字）: 一般的な応募フォーム向け
4. **詳細版**（800-1000文字）: 詳細記入欄向け

## 出力フォーマット

```markdown
# 案件応募PR文

## マッチング分析

### 募集要項サマリ
| 項目 | 内容 |
|------|------|
| ポジション | ... |
| 必須スキル | ... |
| 歓迎スキル | ... |

### マッチ度評価
| カテゴリ | マッチ度 | 該当経験 | 出典（職務経歴書の該当箇所） |
|----------|----------|----------|----------------------------|
| 必須スキル | 高/中/低/該当なし | 具体的な経験 | 「職務経歴 > プロジェクト名」等 |
| 歓迎スキル | 高/中/低/該当なし | 具体的な経験 | 出典なければ「該当なし」 |

### 強調ポイント
- 募集要項に最もマッチする経験・スキル

### ギャップ・補足説明が必要な点
- 不足スキルとカバー方法

---

## PR文（バリエーション別）

### 超短縮版（100文字）
（一言で強みを伝える）

### 短縮版（200文字）
（コンパクトに実績と意気込み）

### 標準版（400-600文字）
（応募フォーム向け、構造化されたPR）

### 詳細版（800-1000文字）
（詳細な実績と意気込み）

（案件特性に応じた調整ポイント）
```

## PR文作成の原則

### 構成（STAR+意気込み）
1. **Situation**: 背景・環境
2. **Task**: 担当した役割・課題
3. **Action**: 具体的なアクション
4. **Result**: 定量的な成果
5. **意気込み**: この案件で活かしたいこと

### 効果的なPR文のポイント
- 募集要項のキーワードを自然に織り込む
- 定量的な実績を含める（人数、期間、規模）
- 「御社で」「この案件で」の具体的な貢献イメージ
- 謙虚さと自信のバランス

### NGパターン
- 抽象的な表現のみ（「頑張ります」「貢献します」）
- 募集要項と無関係な経験の羅列
- 長すぎる自己紹介（本題に入るまでが長い）
- **職務経歴書にない経験・スキルの記載（創作）**
- **「〜もできる」「〜にも対応可能」等の推測表現**
- **数値・期間・規模の誇張**
