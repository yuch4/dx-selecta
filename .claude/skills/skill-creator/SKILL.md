# Skill Creator

効果的なスキルを作成するためのガイド。

**必読**: 作成前に `.claude/rules/skills-vs-commands.md` を参照し、SkillとCommandのどちらが適切か判断すること。

## スキルとは

スキルは、専門知識、ワークフロー、ツールを提供することでClaudeの能力を拡張するモジュール。特定のドメインやタスク向けの「オンボーディングガイド」として機能。

### スキルが提供するもの

1. 特化したワークフロー - 特定ドメイン向けの多段階手順
2. ツール統合 - 特定ファイル形式やAPIとの連携手順
3. ドメイン専門知識 - 会社固有の知識、スキーマ、ビジネスロジック
4. バンドルリソース - スクリプト、参照資料、アセット

## 核心原則

### 簡潔さが鍵

コンテキストウィンドウは共有資源。Claudeがすでに知っていることは書かない。「Claudeにこの説明は本当に必要か？」を常に問う。

### 適切な自由度

| 自由度 | 使用場面 | 形式 |
|--------|----------|------|
| 高 | 複数アプローチが有効 | テキスト指示 |
| 中 | 好ましいパターンがある | 擬似コード |
| 低 | エラーが致命的 | 具体的スクリプト |

## スキルの構造

```
skill-name/
├── SKILL.md (必須)
│   ├── YAML frontmatter（name, description, allowed-tools, user-invocable）
│   └── Markdown本体
└── references/      # 参照ドキュメント（オプション）
```

### SKILL.md フロントマター

```yaml
---
name: skill-name                    # 必須。英小文字・数字・ハイフン
description: 何をする / いつ使う     # 必須。トリガーワードを含む
allowed-tools: Read, Grep, Glob     # オプション
user-invocable: true                # オプション
---
```

### references/

参照資料用。SKILL.mdから明確にリンクする。

- **いつ使うか**: Claudeが作業中に参照すべきドキュメント
- **例**: スキーマ、APIドキュメント、ドメイン知識、ポリシー
- **ベストプラクティス**: 大きなファイル（>10k語）には目次を含める

## スキル作成プロセス

### Step 1: 具体例で理解
- 「このスキルはどう使われるか？」
- 「ユーザーは何と言ってトリガーするか？」

### Step 2: 再利用可能コンテンツを計画
- scripts/ - 繰り返し書き直すコード
- references/ - 参照ドキュメント

### Step 3: スキルを作成
- ディレクトリとSKILL.mdを作成
- フロントマターを書く（descriptionが最重要）
- 本体に手順を書く

### Step 4: テストと反復
- 実タスクで使用
- 問題点を特定
- 改善を実装

## ベストプラクティス

1. **SKILL.mdは500行以下** - 超えたらreferencesに分割
2. **参照は1階層まで** - 深いネストを避ける
3. **descriptionにトリガーワードを網羅**
4. **不要なファイルを作らない** - README.md等は不要
5. **Progressive Disclosure** - 必要時のみ詳細を読み込む
