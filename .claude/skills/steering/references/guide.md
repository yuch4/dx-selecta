# Steering Guide

作業を計画・追跡・振り返りするための詳細ガイド。

## ディレクトリ構造

```
.steering/
├── 20241215-feature-auth/
│   ├── requirements.md
│   ├── design.md
│   └── tasklist.md
└── 20241218-fix-bug/
    ├── requirements.md
    ├── design.md
    └── tasklist.md
```

## タスクリストのルール

### タスクの粒度
- 1タスク = 1-2時間以内で完了できるサイズ
- 大きすぎる場合は分割する
- 依存関係を考慮して順序を決める

### ステータス管理
- `[ ]` - 未着手
- `[x]` - 完了
- `[>]` - 進行中（オプション）
- `[-]` - スキップ/中止（理由を記載）

### 更新タイミング
- タスク開始時にステータス更新
- タスク完了時にチェック
- 新しいタスクが見つかったら追加
- スコープ外と判断したら別セッションに移動

## 振り返りの書き方

### 完了後に記録すること
1. **何がうまくいったか**
2. **何が困難だったか**
3. **次回への教訓**
4. **残課題・フォローアップ**

### 例

```markdown
## 振り返り

### うまくいったこと
- Server Actionsへの移行がスムーズだった
- テストを先に書いたことで仕様が明確になった

### 困難だったこと
- 型エラーの解消に時間がかかった
- 既存コードの理解に手間取った

### 次回への教訓
- 事前に既存コードをもっと読むべき
- 型定義を先に設計すると効率的

### 残課題
- パフォーマンス最適化（P2）
- ドキュメント更新（P1）
```
