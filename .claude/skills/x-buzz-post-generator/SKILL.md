# X 投稿ジェネレーター

2つのモードで目的に応じた投稿を生成する。

---

## 新アルゴリズム原則（2025年1月〜）

**GrokがAIが全投稿を解析し「本当に面白いか」を判断する時代**。以下の原則を全投稿に適用する。

### リーチ枠の概念

> 「投稿するたびに、その日はあなたのリーチの一部が消費される」— Nikita Bier（X Product）

- 平均ユーザーは1日20〜30投稿しか見ない
- 大量投稿・大量リプライ＝リーチ枠の浪費
- **質の高い投稿に集中**することでリーチを最大化

### Grok評価基準

| 評価ポイント | 説明 |
|-------------|------|
| コンテンツの面白さ | 情報価値・独自性・深さ |
| 文脈の充実度 | 「なぜ重要か」が明確か |
| 独自の視点 | 単なる情報共有でなく考察があるか |
| 画像・動画の質 | 視覚情報の付加価値 |

### 3つの必須要素

**全投稿に以下の要素を含めること**（リンクのみ投稿は厳禁）：

1. **なぜこの情報が重要なのか** — 背景や意義を説明
2. **どこが面白いのか** — 具体的な注目ポイント
3. **自分はどう思ったのか** — 独自の考察・意見

ユーザーの入力内容から適切なモードを自動判定する：

| モード | 判定条件 | 目的 |
|-------|---------|------|

## 入力フロー（共通）

### Step 1: トピックの受け取り
ユーザーから話題・情報を受け取る

### Step 2: 意見・スタンスのヒアリング

```
投稿に込める「あなたの視点」を教えてください

【スタンス】どれに近いですか？
□ ポジティブ（期待・賞賛・応援）
□ ネガティブ（懸念・批判・警鐘）
□ 中立・分析的（客観的に整理）
□ 驚き・衝撃（予想外だった）
□ 共感・あるある（自分も経験ある）

【感想・意見】（箇条書き・断片的でOK）
```

### Step 3: 意見の補足・整理

ユーザー入力が不完全な場合、適切に補足提案する

## バズモード

話題のトピックから5種類のバズる投稿文を生成する。

### ワークフロー

1. ユーザーから話題のトピック・文脈を受け取る
2. **意見・スタンスをヒアリング（不完全なら補足提案）**
3. トピックの本質と拡散ポイントを分析
4. 23種類のパターンから最適な5種類を選択
5. 選択したパターンで投稿文を生成（意見を組み込み）
6. 各投稿にハッシュタグ案を付与

### 投稿パターン一覧（23種類）

詳細なテンプレートは [references/patterns.md](references/patterns.md) を参照。

**基本パターン（5種）:** 共感・あるある型、驚き・意外性型、物語・体験型、問いかけ・議論喚起型、ユーモア・ネタ型

**体験・変化系（5種）:** 逆張り＋体験談、時系列変化、やめたら良くなった系、ビフォーアフター、ストーリーフック

**リスト・比較系（5種）:** 5選リスト、ランキング、比較表、よくある勘違い、やるな・やれ

**意見・本音系（4種）:** 告白・本音、引用＋意見、逆張り意見、気づきシェア

**情報・行動系（4種）:** 衝撃の数字＋解説、衝撃の真実、質問→回答、行動喚起

### 作成ルール（バズモード）

- 文字数は300字程度を目安（日本語基準）
- 絵文字は控えめに（0-2個程度）
- こまめに改行を効果的に使う
- 句読点は使わない
- 炎上リスクのある表現は避ける
- 特定の個人・団体を攻撃しない
- **必ず3つの必須要素（なぜ重要/どこが面白い/自分の意見）を含める**
- **リンクだけの投稿は絶対に生成しない**

## アップデート共有モード

VSCode、GitHub、SaaS等の製品アップデート情報をわかりやすく構造化して共有する。

### アップデート共有パターン（8種類）

詳細は [references/update_patterns.md](references/update_patterns.md) を参照。

| パターン | 用途 |
|---------|------|
| ハイライト型 | 注目機能1つを深掘り |
| まとめリスト型 | 複数機能を一覧化 |
| ビフォーアフター型 | 変更点の比較 |
| ワンポイント型 | 小さいけど便利な機能 |
| スレッド型 | 大型アップデート |
| ユースケース型 | 実際の使い方 |
| 注意喚起型 | 破壊的変更・非推奨化 |
| 比較型 | 既存機能との違い |

## 投稿最適化チェックリスト

生成した投稿を以下の観点で最終チェック：

### Grok評価チェック
- [ ] リンクだけでなく文脈・考察がある
- [ ] 「なぜ重要か」が明確
- [ ] 「どこが面白いか」を具体的に指摘
- [ ] 「自分の意見・感想」が含まれている
- [ ] 画像・図解を添付すべき内容か確認

### 投稿戦略アドバイス

```
投稿戦略メモ
- 推奨添付：[画像/図解/スクリーンショット/GIF/動画]
- 投稿タイミング：平日12-13時、20-22時が狙い目
- フォローアップ：価値あるリプライには丁寧に返信
- 注意：大量リプライはリーチ枠を浪費するため控える
```
